## 已随仓库生成的素材（可直接用）

仓库内 **`images/*.jpg`** 均由脚本 **`scripts/build_assets.py`**（Pillow）本地绘制，为**原创抽象视觉**，用于占满所有原先的空位；**不是**各公司/学校的官方商标或真实产品截图。

| 类型 | 文件 |
|------|------|
| 站点图标 | `images/favicon.svg` |
| 导航旁「公司感」抽象标 | `images/icons/mark-fluxvita.svg`、`mark-jovida.svg`、`mark-greenplatter.svg`、`mark-academic.svg` |
| 光栅横幅 / 卡片 / 胶片等 | 见下表「建议文件名」列，已全部生成同名 JPG |

**法律说明**：Fluxvita、Jovida、GreenPlatter、密歇根大学等 **正式 Logo / 校徽 / 产品 UI 截图** 受版权与商标约束，**不能**由我擅自从网上扒下来塞进你的仓库。当前 SVG 为**几何抽象标**，仅作视觉联想；对外正式场合请换 **官方品牌包** 或你方设计稿。

**「本人照片」**：`hero-portrait.jpg` 当前为 **YKL 字母标 + 剪影占位**，并非真人肖像。请用你的高清肖像 **覆盖该文件**（保持文件名不变最省事），或改 HTML 中的 `src`。

**重新生成所有 JPG**（例如改配色后）：

```bash
python academic-site-framework/scripts/build_assets.py
```

## 执行顺序建议

1. 先拍或约拍 **IMG-01 主肖像**（整站气质锚点）。  
2. 再补 **三张卡片图**（产品 / 代谢 / 学校）与 **work-header**（二级页第一眼）。

4. 其余按上线优先级迭代。

## 技术说明

- 占位依赖 CSS `:not(:has(> img))`；放入 `<img src="images/xxx.jpg" alt="…" />` 后占位自动消失。  
- 裁切：`object-fit: cover`，重要内容请留在**画面中心 60%** 内。  
- 大图请压缩（如 WebP 另存 + 改扩展名引用，或 TinyPNG），避免首屏过慢。
