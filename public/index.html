初始设置

第一次跑这个项目？跟着做下面 2 步就能用了。卡在哪一步看 AGENTS.md。

1

填三个凭据

所有字段只存到本机的 config/local.json（git 忽略），不发给我们，不进仓。

2

登录 YouTube 账号

点下面按钮会跳转到 Google 登录页。授权完成后浏览器会自动跳回这里，token 存到本机 credentials/token.json（git 忽略）。

⚠️ 你的 Gmail 必须先被加进 GCP 同意屏幕的「测试用户」名单，否则 Google 会直接拒。

不想用向导？

把 YOUTUBE_OAUTH_CLIENT_ID / YOUTUBE_OAUTH_CLIENT_SECRET / GEMINI_API_KEY 写到 .env（参考 env.example），然后 npm run oauth 走 CLI 流程。重启 server。

合规风险（请阅读）

批量自动发评论（哪怕评论由 LLM 生成）撞 YouTube ToS 的「自动化/反垃圾」条款。 Google 自动反垃圾系统可能将整条评论自动隐藏，或对账号施加限速、降权乃至永久封禁。 本工具采用「半自动」流程：管道生成候选 → 你逐条点发送，但**最终发送行为仍由本工具用 OAuth token 调用 commentThreads.insert 完成**， 归责仍在使用者。请评估业务必要性、量级与频次。
